# 示例测试图片说明

## 1. 测试图片准备建议

为了更好地展示项目效果，建议准备 4 到 6 张不同类型的车辆图片，覆盖以下场景：

- 正面或侧前方较清晰车辆图片
- 光线正常、背景较简单的图片
- 背景稍复杂的图片
- 距离略远、车牌区域较小的图片
- 角度略有倾斜的图片

## 2. 推荐测试图片分类

### A. 标准样例图

特点：

- 光线较好
- 车牌清晰
- 背景简单

用途：

- 用于展示系统在理想情况下的处理流程
- 适合做 README 首页截图

### B. 复杂背景图

特点：

- 背景中存在较多纹理或车辆周边元素
- 车牌不是画面中唯一明显区域

用途：

- 用于展示轮廓筛选和候选区域定位能力
- 面试时可以说明项目的局限和优化方向

### C. 倾斜角度图

特点：

- 车辆角度不是完全正对镜头
- 车牌区域有一定透视变化

用途：

- 用于展示项目在非理想条件下的处理效果

### D. 远距离图

特点：

- 车牌区域较小
- 识别难度更高

用途：

- 用于说明当前项目的适用范围和后续改进空间

## 3. 命名建议

为了后续展示和管理方便，建议统一命名：

```text
sample_01_clear.jpg
sample_02_complex.jpg
sample_03_tilt.jpg
sample_04_far.jpg
sample_05_night.jpg
```

## 4. 展示时建议保留的图片材料

建议每张测试图片至少保留以下 3 类结果：

- 原始图片
- 处理后图片
- 车牌候选区域截图

如果后续做项目展示页或 GitHub 页面，可以按“原图 / 处理图 / 候选区域图”三联图方式排版。

## 5. 面试展示建议

面试时优先展示两类图片：

- 一张效果较好的标准图
- 一张效果一般但能说明问题的复杂图

这样既能体现项目完成度，也能说明你对项目边界和优化方向有清楚认识。
