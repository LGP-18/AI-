# 面试时怎么演示这个作品

## 1. 演示目标

面试演示的重点不是证明你做了一个商用级系统，而是让面试官快速理解：

- 你做过一个完整的计算机视觉应用项目
- 你理解图像处理的基本流程
- 你能把处理结果做成可展示的小系统

## 2. 最佳演示时长

建议控制在 `3 到 5 分钟`，不要讲太久。

## 3. 推荐演示顺序

### 第一步：一句话介绍项目

可以这样说：

“这是我做的一个基于 OpenCV 的车牌识别与信息管理系统，主要面向车辆出入管理场景，重点完成图像预处理、车牌候选区域定位、结果展示和信息查询这几个功能。”

### 第二步：讲项目目标

可以这样说：

“我做这个项目的目的是把图像处理和结果展示串成一个完整流程，而不是只停留在本地跑图像脚本。”

### 第三步：展示页面

优先展示：

- 上传图片页面
- 本次识别结果
- 历史记录表

讲解重点：

- 可以上传车辆图片
- 系统会做基础图像处理
- 会输出处理后的图和候选区域图
- 结果会存到数据库，页面中可以查询

### 第四步：讲技术流程

建议直接按这个顺序讲：

1. 图片上传
2. 灰度化、去噪、边缘检测
3. 轮廓筛选
4. 车牌候选区域提取
5. 结果整理
6. Flask 展示
7. SQLite 存储

### 第五步：说明你负责的部分

可以这样说：

“这个项目中我主要负责图片数据整理、图像预处理、候选区域提取，以及 Web 端展示和结果管理这部分实现。”

### 第六步：主动说局限

这一步很重要，会显得你更真实。

可以这样说：

“这个项目当前重点是基础图像处理和流程展示，车牌文本识别部分还是演示级实现。后续如果继续完善，可以接入 OCR 或检测模型来提升准确性。”

## 4. 面试官如果追问，你可以怎么答

### 如果问：这个项目难点是什么？

回答思路：

- 车牌区域定位不是每张图片都稳定
- 背景复杂时轮廓筛选效果会受影响
- 所以后续还有优化空间

### 如果问：为什么不用深度学习模型？

回答思路：

- 当前项目定位是大学生计算机视觉应用实践
- 重点是先把图像处理、区域提取、结果展示和系统集成流程跑通
- 后续如果深入做，可以继续引入检测模型

### 如果问：你最大的收获是什么？

回答思路：

- 不只是学会了图像处理方法
- 更重要的是理解了如何把视觉处理做成一个可展示、可查询的小系统

## 5. 演示时不要这样做

- 不要上来就堆很多算法名词
- 不要把演示讲成论文答辩
- 不要夸大成“高准确率识别系统”
- 不要说自己做了没做过的模型训练

## 6. 一段可直接背的简版话术

“这是我做的一个基于 OpenCV 的车牌识别与信息管理系统，主要面向车辆出入管理场景。项目里我主要完成了图片数据整理、图像预处理、车牌候选区域提取，以及识别结果展示和查询功能。技术上主要用到了 Python、OpenCV、Flask 和 SQLite。这个项目让我比较完整地理解了从图像处理到结果展示、再到基础系统集成的应用落地流程。”
